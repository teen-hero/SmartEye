#### 1.2项目特色

​		SmartEye（慧眼仓储）是一款综合的智能仓储系统，集PC端、手机端、设备终端于一体，致力于为企业提供仓储智能化升级集成解决方案。

​		区别于其他同类系统：

​		SmartEye自主设计的业务逻辑，抽取于企业实际生产现状，结合IE、LE等专业知识进行再设计，最终以最贴合实际、最节省、最高效的业务流程开发系统；

​		SmartEye自主设计的多策略上架建议、出库建议、送料建议等智能算法对企业生产起到巨大帮助；

​		SmartEye自主设计的库存模型具有高度适配性，支持虚拟分库，极大的降低企业应用的成本。

​		SmartEye平台设计灵活，可扩展、可移植、可应对高并发需求。采用微服务架构，将功能分解到各个离散的服务中以实现服务解耦，降低系统的耦合性，同时每个微服务能够独立地进行开发、管理和迭代，降低系统维护的难度。

###### ②上架货物

![image-20221026154325427](https://teen-hero.github.io/images/image-20221026154325427.png)

**上架货物：**主要负责对入库时的整批货物进行管理，保留入库单元所属的货物批次，并从整批货物的角度进行上架管理。即该批次货物包含了哪些入库单元，当所包含的所有入库单元均完成上架时，整批货物才可完成上架操作。

**具体功能如下：**

- 点击 [详情] 操作，可查看该批次货物所包含的入库单元信息

<img src="https://teen-hero.github.io/images/image-20221026155501636.png" alt="image-20221026155501636" style="zoom: 33%;" />

- 点击 [上架] 操作，可对整批货物主动进行上架确认。若该批次货物含有未上架的入库单元，则无法完成上架。
  - 该操作主要是由系统自动监测所含入库单元是否全部上架，是则系统自动确认该批货物已上架

![image-20221026155532225](https://teen-hero.github.io/images/image-20221026155532225.png)

**PS：**该模块意义在于，实际入库场景中，不可避免的会将整批货物进行拆解分开上架，而货物拆解后会使得对货物批次的追溯性断联，而且无法有效确认某批次货物包含的入库单元信息以及它们是否全部完成上架。

##### 3.1.4出库

###### ①新建出库单

![image-20221026164305021](https://teen-hero.github.io/images/image-20221026164305021.png)

**新建出库单：**主要负责货物的出库，主要分为物料申请单出库、手动出库，来应对分库物料申请、生产出库等业务场景。

**具体功能：**

- 物料申请单出库，主要处理分库的物料申请，点击分库物料审批模块中的 [处理并生成出库单] 操作，会根据当前物料申请自动填写物料申请单出库的物料申请单ID，之后用户补充其他出库信息，最终形成出库单。

![image-20221026165737772](https://teen-hero.github.io/images/image-20221026165737772.png)

![image-20221026171138596](https://teen-hero.github.io/images/image-20221026171138596.png)

- 手动出库，主要处理生产出库，需要用户填写表单形成出库单

**PS：**针对物料申请出库、手动出库，SmartEye设计了出库推荐功能，根据出库需求及库存信息系统自动生成推荐出库单，企业可按需选择是否开启该功能。其中出库推荐分为两种策略：均匀策略、穷尽策略。

**解释：**

均匀策略：根据出库需求，由拥有需求货物的各个货架出库一定数量的货物，拼凑出指定的需求数量，最终使得各个货架剩余的需求货物数量尽量均衡。

穷尽策略：根据出库需求，由拥有需求货物的货架依次穷尽出库，直到拼凑出指定的需求数量为止，最终使得某一些货架对需求货物的库存为零。

### **4.库存模型**

#### 4.1整体模型

SamrtEye库存模型整体如下

![image-20221026114502513](https://teen-hero.github.io/images/image-20221026114502513.png)

​		库存模型整体由总库、分库组成，分库归属于各个分厂区（A厂区、B厂区..）。

​		总库负责采购货物的入库处理，对接企业外部；

​		分库负责向总库进行物料申请、处理车间产线的叫料等（更像是一个大型的暂存区），对接企业内部。

#### 4.2设立分库的意义

![image-20221027145532497](https://teen-hero.github.io/images/image-20221027145532497.png)

​		总库和实际生产线之间设立分库，分库主要负责所在分厂区的货物管理，向上对接总库，向下对接生产线。

​		分库的意义在于：避免零散、分散、数量较多的生产线直接向总库获取库存，而是经分库初步管理、汇集零散的生产线需求集中向总库进行物料申请，避免了生产库存的管理混乱（分库更像一个中间件，缓存库存）

#### 4.5模型优势

**优势：**

- 库存模型高度适配性
  - SamrtEye所设计的库存模型，可由真实库存仓库构建而成，且总分库分区划分可跟随企业实际库存结构动态组合完成，分库的划分也可根据实际情况进行适配。
  - 同时，为缩减企业应用SamrtEye而产生的成本，SamrtEye支持虚拟分库库存，即企业无需设立分库，使用现有的库存仓库，划分出分库专属区域即可完美适配系统。
- 系统高度适配性
  - SamrtEye虽设计了原生的库存模型，但却不局限于原生库存模型，SamrtEye同时可以支持其他不同的库存模型，只要企业有仓库、有货架，SamrtEye都可进行适配。
